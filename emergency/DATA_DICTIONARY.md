# FAICC PRO 数据字典与业务定义说明 (v2.0)

本系统为「消防 AI 智能指挥决策系统」，旨在通过多维度数据聚合，为应急指挥官提供实时、科学的决策依据。

## 1. 核心指标定义 (KPIs)

| 指标名称 | 业务定义 | 数据来源 | 更新频率 |
| :--- | :--- | :--- | :--- |
| **当日累计警情** | 自今日 00:00 起，系统接收到的所有有效火灾报警总数。 | `fire_events` 表总记录数 | 实时 (30s 轮询) |
| **当前处置中** | 状态标记为 `processing` 的警情数量，代表正在消耗救援力量的事件。 | `fire_events.status` | 实时 |
| **在岗消防员** | 正在参与一线救援的消防人员总数。 | 基于 `severity` 等级模拟计算 | 实时 |
| **出动车辆** | 正在执行任务或前往现场的消防车总数。 | 基于 `severity` 等级模拟计算 | 实时 |

## 2. 专业图表业务含义

### 2.1 警情等级分布统计 (Severity Level Stats)
- **图表类型**：专业柱状图 (Bar Chart)
- **业务定义**：展示 L1 至 L5 不同严重程度的警情分布。
- **决策参考**：若 L4/L5 占比较高，需考虑启动城市级一级响应并请求跨区支援。

### 2.2 核心场景风险分布 (Scene Analysis)
- **图表类型**：环形图 (Donut Chart)
- **业务定义**：基于 AI 语义识别，将警情归类为「人员密集区」、「易燃易爆区」、「普通居民区」等。
- **决策参考**：易燃易爆区比例升高时，应重点调配特种消防器材（如泡沫车）。

### 2.3 区域火险热力态势 (Tactical Heatmap)
- **图表类型**：笛卡尔坐标系热力图 (Heatmap)
- **业务定义**：展示城市地理空间内的警情密集程度。
- **决策参考**：热力值极高的区域（红色）为当前火险爆发中心，需优化该区域的消防站点覆盖。

### 2.4 警情变化趋势 (Time Series Trend)
- **图表类型**：平滑折线图 (Line Chart)
- **业务定义**：近 8 小时内每小时新增警情数量的变化曲线。
- **决策参考**：识别警情高峰时段（如深夜或午间），优化值班排班计划。

### 2.5 智能体协同效能指标 (Agent Metrics)
- **图表类型**：仪表盘 (Gauge) + 指标卡
- **指标说明**：
  - **AI 决策准确率**：基于历史反馈修正后的 AI 建议采纳率。
  - **平均响应耗时**：从接警到首支力量到达现场的平均分钟数。
  - **资源负荷率**：当前动用的力量占总编制力量的比例。

## 3. 使用手册

### 3.1 环境准备
1. 确保已安装 Python 环境。
2. 启动后端 API 服务：
   ```powershell
   python FireAgent_Competition/mock_agents.py
   ```
3. 服务运行在 `http://127.0.0.1:8000`。

### 3.2 大屏操作指南
- **标准分辨率**：推荐在 1920×1080 环境下全屏访问（按 F11）。
- **实时刷新**：系统默认每 30 秒自动刷新数据，您也可以通过顶部的「刷新」状态确认连接。
- **多维筛选**：
  - 使用左上角下拉框可筛选「近 1 小时」至「全量」的数据。
  - 支持按「行政区域」（如朝阳区）进行局部态势分析。
- **报告导出**：
  - 点击右上角 **PNG**：直接下载当前大屏的高清截图。
  - 点击右上角 **导出报告**：生成 PDF 格式的决策快报，便于离线传阅。

---
*注：本大屏数据受多智能体（气象、交通）实时修正，所有地理坐标及资源消耗均为 AI 模拟测算。*
